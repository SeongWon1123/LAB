# Product Requirements

## One-Line Pitch

Meet a tiny pixel friend that feels like a forgotten desk toy rediscovered inside your phone.

## MVP Goal

Pocket Memory Pet must let a user hatch, name, care for, and revisit an original pixel pet without any server dependency.

## Core Flow

1. Open app.
2. Read a short onboarding screen.
3. Name the pet.
4. Watch the hatch transition.
5. Land on the home screen.
6. Use three buttons to feed, snack, play, clean, sleep, wake, inspect status, and read diary entries.
7. Close and reopen the app.
8. Pet state updates from UTC elapsed time and persists locally.

## Functional Requirements

- Pet naming.
- Hatch flow.
- Home screen with original retro device shell.
- LCD-style pixel display.
- Three-button controls.
- Hunger, happiness, cleanliness, energy, health, discipline, age, care mistakes, and growth stage.
- Meal, snack, play, clean, sleep, wake actions.
- Automatic diary entries for important care and growth events.
- Local notification permission explanation and settings.
- Hive CE persistence.

## Non-Goals

No cloud storage, account system, ads, payments, social graph, chat, ranking, AR, widgets, UGC, or community features in MVP.

## Success Criteria

- User meets the pet within one minute.
- Three buttons are enough for core care.
- State persists after restart.
- Time changes the pet naturally.
- Growth diary entries accumulate automatically.
- App remains usable when notification permission is denied.
